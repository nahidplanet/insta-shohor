আমরা জানি javascript কে run করার জন্য ব্যবহার করা হয় ক্রোমের V8 ইঞ্জিন। এছাড়াও বিভিন্ন ব্রাউজার আলাদা আলাদ ইঞ্জিনের সাহায্য নিয়ে থাকে।আমরা যখন কোনো javascript কোড কে run করায় তখন call stack এর মধ্যে গ্লোবাল execution context হিসেবে main() ফাংশনকে কল করে।তারপর javascript synchronised পদ্ধতিতে source কোডের ফাংশন গুলোকে এক এক করে callstack এ তুলতে থাকে এবং প্রিন্ট করতে থাকে। এই প্রক্রিয়া টি last in fast out পদ্ধতিতে হয়ে থাকে। একএক করে প্রিন্ট হয়ে callstack. থেকে লাস্ট থেকে ফাস্টের দিকে একটি করি করে প্রিন্ট করে out হতে থাকে। তবে synchronous. এর জায়গায় asynchronous ফাংশন এলে javascript এক এক করে callstack এ ফাংশন গুলো তুলে সে বসে থাকে না বরং asynchronous ফাংশনকে ব্রাউজারে ক্ষেত্রে web api এবং node এর ক্ষেত্রে c++ apiapi এর কাছে ছেড়ে দিয়ে বাকি ফাংশনগুলো execution করে। callstack এর সব কাজ শেষ হলে ঐ asynchronous ফাংশনকে web api/c++ api থেকে callback queue এর কাছে পাঠিয়ে দেয়। এই callback queue থেকে callstack. এ একএক একরে তুলতে ও প্রিন্ট করতে Event Loop কাজ করে।

কম্পিউটার আমাদের কোড বুঝতে পারে না। আবার আমরা কম্পিউটারের কোড বুঝতে পারি না। কম্পিউটার এ আমরা যে js কোড লিখি তা ব্রাউজারের বা node এর v8 ইঞ্জিন পার্স করে একধরনের সিন টেক্সট ট্রি আকারে সাজায়। তারপর interpreted o compiled করার কাজটি JIT(just in time) এর মাধ্যমে byte code এ রুপান্তর করে। ফলে কম্পিউটার তা বুঝতে পারে। ইন্টারপ্রেট লাইন বাই লাইন কোড read করে যা ডিবাগ হেল্পফুল হলেও অনেক স্লো প্রসেস, আবার কম্পাইলার পুরো কোড read করে বলে ডিবাগ করাটা কষ্টসাধ্য ব্যাপার। তাই এই সমস্যার সমাধান করে JIT। ফলে javascript একই সাথে যেমন ফাস্ট run করে, তেমনি ডিবাগ করতেও সুবিধা।

Web storage Api আমাদের ব্রাউজারে ডাটা read,Write,update, delete করতে সাহায্য করে থাকে। Web storage এর প্রধান ২টি জিনিস হলো
1. Local Storage
2.Session Storage
Local Storage : Local Storage এর সাহায্য ডাটা Store করলে তা ব্রাউজার close করার পড়েও থেকে যায়। ফলে user অনেক দিন পূর্বের ডাটা তার ব্রাউজার থেকেই পেয়ে থাকে।
Session Storage : Session storage এর ডাটা স্থানীয় নয় বরং কিছু সময় পর্যন্ত তা জমা থাকে ব্রাউজারে। ব্রাউজার close করার সাথে সাথে তা ব্রাউজার থেকে ডিলিট হয়ে যায়।

Web storage এর আরেকটি গুরুত্বপূর্ণ বিষয় হলো cookies।
cookies কোন নির্দিষ্ট সময় পর্যন্ত user এর ডাটা সেভ করে রাখতে পারে। ব্রাউজার কোনো সার্ভারে রিকুয়েষ্ট পাঠাতে গিয়ে cookies এর মাধ্যমে কিছু ডাটা ব্রাউজারে জমা থাকে আবার সেই ডাটা সার্ভার চাইলে জমা রাখতে পারে এতে করে পরবর্তী তে সার্ভার বুঝতে পারে কোন user রিকুয়েষ্ট পাঠিয়েছে। বিভিন্ন লগিন এক্সেস এ cookies এর ব্যবহার দেখা যায়। ফলে user কে বার বার ইনফরমেশন দিয়ে লগইন করতে হয় না।
